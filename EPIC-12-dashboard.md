# EPIC-12: Dashboard Build (Umbrella)

**Status:** Not Started
**Dependencies:** EPIC-01, EPIC-02

## Overview

Umbrella epic that restructures the dashboard build into a phased approach. Replaces the fine-grained sequential dependency chain of EPICs 03–11 with a single epic containing 12 issues across 4 phases: foundation → vertical slices → Metabase → AI.

Statement processing pipeline (EPIC-03) is intentionally excluded — it's ingestion infrastructure, not dashboard functionality. Categorization rules engine (EPIC-05) is also deferred — only Category CRUD is included here.

## Phase 1 — Foundation

### 12.1: Schema Extension — New Models & Transaction Fields

**Scope:** Add all remaining database models and extend the Transaction model with feature-linking fields. Single Prisma migration.

**Depends on:** EPIC-02 (Transaction model must exist)
**Reference:** EPIC-06, EPIC-07, EPIC-08, EPIC-09

#### New Enums

| Enum | Values |
|------|--------|
| `RecurringFrequency` | `MONTHLY`, `YEARLY`, `WEEKLY` |
| `RecurringType` | `SUBSCRIPTION`, `BILL`, `INSTALLMENT` |
| `BudgetStatus` | `ACTIVE`, `PAUSED` |

#### New Models

| Model | Key Fields |
|-------|-----------|
| `Budget` | id, category_id (FK), amount (Decimal(12,2)), period (String, nullable — null = default template, "YYYY-MM" = monthly override), status (BudgetStatus, default ACTIVE), created_at, updated_at |
| `RecurringItem` | id, name, amount (Decimal(12,2)), frequency (RecurringFrequency), next_date (Date), type (RecurringType), category_id (FK → Category, nullable), account_id (FK → Account, nullable), created_at, updated_at |
| `InstallmentGroup` | id, description, total_amount (Decimal(12,2)), installments_total (Int), installments_paid (Int), start_date (Date), category_id (FK → Category, nullable), account_id (FK → Account, nullable), created_at, updated_at |
| `SavingGoal` | id, name, target_amount (Decimal(12,2)), deadline (Date, nullable), linked_account_id (FK → Account, nullable), created_at, updated_at |

#### Transaction Extensions

Add 5 fields to the existing `Transaction` model:

| Field | Type | Notes |
|-------|------|-------|
| `is_recurring` | Boolean | Default false |
| `recurring_item_id` | UUID, nullable | FK → RecurringItem |
| `installment_group_id` | UUID, nullable | FK → InstallmentGroup |
| `is_savings_transfer` | Boolean | Default false |
| `saving_goal_id` | UUID, nullable | FK → SavingGoal |

#### Constraints

- `Budget`: unique on `(category_id, period)` — one budget per category per period. Requires two constraints: (1) standard unique on `(category_id, period)` for non-null periods, (2) partial unique index on `category_id WHERE period IS NULL` to enforce one default template per category (PostgreSQL treats NULLs as distinct in unique constraints)
- `SavingGoal.linked_account_id` is optional — goals can exist without a dedicated savings account
- Transaction FKs to `RecurringItem`, `InstallmentGroup`, and `SavingGoal` use `onDelete: SetNull` — deleting a parent entity nullifies the FK on linked transactions

#### Prerequisites

- Install `jsonwebtoken` (needed for 12.9 Metabase JWT signing)
- Install `@anthropic-ai/sdk` (needed for 12.12 Claude API streaming)

#### Files

- `app/prisma/schema.prisma` — add 3 enums, 4 models, extend Transaction
- Migration file (auto-generated by `prisma migrate dev`)

#### Acceptance Criteria

- [ ] Migration applies cleanly on top of EPIC-02's schema
- [ ] All 4 new models created with correct field types and defaults
- [ ] Transaction model has 5 new fields with proper FK relations
- [ ] `Budget` unique constraint on `(category_id, period)` enforced
- [ ] `npx prisma generate` produces client with all new types
- [ ] Existing tests still pass

## Phase 4 — AI

### 12.12: AI Financial Insights Chat

**Scope:** Chat interface for querying financial data using Claude API with streaming responses.

**Depends on:** 12.1, 12.3 (needs transaction data to query)
**Reference:** EPIC-11

#### Files

- `app/lib/schemas/insights.ts` — Zod schema for chat messages (role + content)
- `app/app/api/insights/route.ts` — POST (accepts conversation history, fetches relevant data from DB, streams Claude response)
- `app/app/(dashboard)/insights/page.tsx` — Chat UI with message history
- `app/components/chat/InsightsChat.tsx` — Chat component with input, message bubbles, streaming indicator
- `app/lib/schemas/insights.test.ts` — Schema tests
- `app/app/api/insights/route.test.ts` — Endpoint tests

#### Environment Variables

- `ANTHROPIC_API_KEY` — Claude API key

#### Architecture

1. User sends message via chat UI
2. API route analyzes the query to determine which data to fetch (transactions, budgets, goals, etc.)
3. Relevant data fetched from PostgreSQL
4. Data + user query sent to Claude API as context
5. Response streamed back to the UI

#### E2E Tests

- `app/e2e/insights.spec.ts` — Chat send/receive, streaming indicator, error state

#### Acceptance Criteria

- [ ] Chat UI renders with message input and conversation history
- [ ] Messages stream in real-time (not wait for full response)
- [ ] Claude receives relevant financial data as context for answering
- [ ] Handles example queries: "How much did I spend on food last month?", "Am I on track for my emergency fund?"
- [ ] Missing `ANTHROPIC_API_KEY` shows clear error, not crash
- [ ] Conversation history maintained within the session
