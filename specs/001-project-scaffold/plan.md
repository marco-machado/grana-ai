# Implementation Plan: Project Scaffold & App Shell

**Branch**: `001-project-scaffold` | **Date**: 2026-03-01 | **Spec**: [spec.md](./spec.md)
**Input**: Feature specification from `/specs/001-project-scaffold/spec.md`

## Summary

Bootstrap the Grana AI application: a Next.js 15 dashboard shell running in Docker with Prisma ORM connected to PostgreSQL 17, three foundational database tables (Account, Source, Category), and a hierarchical category seed. The result is a navigable 8-page dashboard served at `localhost:3000` via `docker compose up`.

## Technical Context

**Language/Version**: TypeScript 5.x (Node 22 Alpine)
**Primary Dependencies**: Next.js 15, React 19, Tailwind CSS 4, Prisma ORM, `tsx` (seed runner)
**Storage**: PostgreSQL 17 (Docker), accessed via Prisma. Schema in `app/prisma/schema.prisma`.
**Testing**: Vitest (single-run mode: `npx vitest run`). Not in scope for this scaffold epic.
**Target Platform**: Docker Compose (self-hosted, macOS/Linux)
**Project Type**: Web application (dashboard)
**Performance Goals**: N/A — single user, development simplicity prioritized.
**Constraints**: Single user, BRL currency only, Docker Compose only, no authentication.
**Scale/Scope**: 1 user, 8 pages (placeholders), 3 DB tables, ~43 seed categories.

## Key Implementation Details

### Dockerfile (Multi-Stage)

Three stages targeting Node 22 Alpine:
1. **dependencies** — `npm ci`, copy `prisma/`, run `prisma generate`
2. **builder** — copy source, `npm run build` (produces `.next/standalone`)
3. **runner** — copy standalone output + static files, run as non-root `node` user

### Docker Compose Updates

The existing `docker-compose.yml` needs updates to the `app` service:
- Add `command` for startup chain: `prisma db push && prisma db seed && node server.js`
- Update `develop.watch` paths for the source code structure
- Ensure `depends_on` with `service_healthy` is maintained (already present)

### Prisma Schema

```prisma
datasource db {
  provider = "postgresql"
  url      = env("DATABASE_URL")
}

generator client {
  provider = "prisma-client"
  output   = "./generated/client"
}

enum AccountType {
  CHECKING
  CREDIT
  SAVINGS
}

enum SourceType {
  EMAIL
  CSV
  API
  MANUAL
}

model Account {
  id         String      @id @default(dbgenerated("gen_random_uuid()")) @db.Uuid
  name       String
  type       AccountType
  currency   String      @default("BRL")
  created_at DateTime    @default(now())
  updated_at DateTime    @updatedAt
  sources    Source[]
}

model Source {
  id         String     @id @default(dbgenerated("gen_random_uuid()")) @db.Uuid
  name       String
  type       SourceType
  identifier String
  account_id String     @db.Uuid
  account    Account    @relation(fields: [account_id], references: [id])
  created_at DateTime   @default(now())
  updated_at DateTime   @updatedAt
}

model Category {
  id         String     @id @default(dbgenerated("gen_random_uuid()")) @db.Uuid
  name       String
  parent_id  String?    @db.Uuid
  parent     Category?  @relation("CategoryHierarchy", fields: [parent_id], references: [id])
  children   Category[] @relation("CategoryHierarchy")
  created_at DateTime   @default(now())

  @@unique([name, parent_id])
}
```

### Next.js 15 Key Patterns

- All page/layout components using `params` or `searchParams` must declare them as `Promise` and `await` them (v15 breaking change).
- Dashboard layout is a Server Component (no `'use client'` needed for static sidebar).
- Route group `(dashboard)` wraps all pages under the sidebar layout without affecting URLs.
